This document defines the scope of features for Learnify NUST, categorized into **MVP (Phase 1)** and **Future Enhancements (Phase 2+)** to ensure focused development and scalability.

---

## Phase 1: MVP Features

### 1. User Authentication
- Login using **NUST university email**
- Secure authentication and session management
- Role-based access (Student, Contributor, Admin)

---

### 2. Course & Section-Based Content
- Courses organized by:
  - Department
  - Semester
  - Course name
- Section-wise breakdown of course topics
- Playlist-based learning structure

---

### 3. Playlist Management
- Embedded YouTube playlists
- Curated playlists mapped to specific course sections
- Metadata support:
  - Title
  - Description
  - Contributor name

---

### 4. Content Contribution (Seniors)
- Seniors can submit:
  - Playlist links
  - Topic descriptions
- Admin approval workflow before public visibility

---

### 5. Content Browsing & Search
- Browse content by:
  - Department
  - Course
  - Section
- Basic search functionality

---

### 6. Admin Dashboard
- Approve or reject submitted content
- Manage users and roles
- Remove outdated or irrelevant content

---

## Phase 2: Future Enhancements

### 1. Faculty Involvement
- Faculty-approved playlists
- Instructor-recommended resources

---

### 2. Personalized Learning
- Progress tracking
- Recommended playlists based on usage

---

### 3. Advanced Search & Filters
- Filters by:
  - Difficulty level
  - Contributor
  - Popularity

---

### 4. Community Features
- Comments and discussions
- Peer recommendations

---

### 5. AI-Assisted Features
- Smart content recommendations
- Automated topic suggestions
- Learning path generation

---

### 6. Monetization & Expansion (Optional)
- Premium content (future)
- Expansion to other universities

---

## Out of Scope (Initial Phase)

- Live video streaming
- Direct file uploads (PDFs, PPTs)
- Mobile application (web-first approach)

---

## Summary

The feature scope ensures:
- A **focused and achievable MVP**
- Clear separation between current and future development
- Scalability without early overengineering
